## 体检脚本

```bash
python scripts/audit_structured_data.py
```

输出文件：

```text
data/reports/structured_data_audit.json
```

该脚本检查：

- `race_results` 中每个场次是否存在 `race_metadata`。
- `race_results` 中每个 runner 是否存在对应 `race_entries`。
- 是否存在重复的赛果 runner。
- 模型训练关键字段是否缺失。
- 采集任务状态是否仍有 `pending` 或 `running`。

## 当前结论

当前本赛季公开元数据已经可以支撑第一版不依赖赔率的模型训练。后续新增赛日采集完成后，应先运行体检脚本，再训练模型或生成预测。

## 待加强

- 继续补全马匹详情页全量历史表现，目前 `horse_profiles` 和 `horse_form_records` 只覆盖少量样本。
- 临场变更页面受 HKJC 当前页面行为影响，历史日期可能返回当前赛日内容；入库时已根据页面实际赛日做保护，但仍需要持续复核。
- 正式数据库落库前，需要把本地 SQLite 表结构映射到 SQLAlchemy/Alembic 的正式 schema。
